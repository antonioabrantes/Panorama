Comentários/Justificativas

1. As alegações do requerente foram analisadas.



2. Em primeiro lugar, em novo entendimento, concorda-se com os esclarecimentos prestados
pelo requerente quanto as questões relativas ao artigo 25 da LPI.



3. Quanto as questões relativas a atividade inventiva,os questionamentos do requerente quanto
a limitação de D1 e D3 não ensinarem autorização do usuario quanto ao download de
atualizações são procedentes. Entretanto entende-se que a definição de uma area de
armazenamento para o download de uma atualização, são uma via comum de escolha por um
tecnico no assunto de tal forma a garantir que os dados referentes ao processo de atualização
não sejam sobre-escritos durante o processo de download. Ora, o documento D1, de forma

ainda que implicitamente, mostra uma area em cache sendo definida para as informações de
atualização, conforme ensina D1(paragrafo 29). Onde as hierarquias de cache são uma forma
usual e comum na tecnica para se definir uma area de armazenamento para atualização, sendo
que entende-se que a caracteristica relativa nas reivindicações independentes principais a
hirerarquia de cache e generica sem qualquer detalhe que o diferencie da função de
armazenamento reservada a atualizações ensinada em D1

3. Por outro lado o documento D2, de forma ainda que implicita ensina que um usuario pode
escolher pelo pacote de atualização desejado para download, conforme ensina D2(pagina
16,linhas 15 a 16).

4. Desta forma entende-se que um tecnico no assunto resolveria o mesmo problema tecnico
associado informações de atualização ao menos pela combinação de D1 e D2.

5. Desta forma quanto ao novo quadro reivindicatorio, na reivindicação 1 o referido meio de
armazenamento legível por computador e mostrado por D1(Figura 1,item 131), em que e
antecipado:

    baixar um ou mais pacotes de atualizações para software em um sistema de computação a
partir de uma fonte remota para o sistema de computação em uma operação em segundo plano
de um ambiente de computação do sistema de computação,conforme D1(paragrafo 30 a 33);
iniciar, pelo sistema de computação, a instalação dos um ou mais pacotes das atualizações
automaticamente na operação em segundo plano antes de confirmar as atualizações, em que
iniciar a instalação dos pacotes inclui construir uma pluralidade de caches de sistema
correspondentes a uma combinação de atualizações predeterminada, cada cache do sistema
tendo uma hierarquia espelhando a hierarquia de um cache do sistema em uso no sistema de
computação,conforme D1(paragrafo 29); mediante aprovação das atualizações, confirmar as
atualizações concluindo a instalação dos pacotes das atualizações,conforme D2(pagina 16).
Desta forma a materia contida na reivindicação 1 esta antecipada pelo estado da tecnica
não apresentando atividade inventiva.



6. Quanto a reivindicação 7, o referido metodo implementado por computador e mostrado por
D1(Figuras 3 a 6B), em que e antecipado: executar uma parte de instalação de um conjunto de
atualizações para o software em um sistema de computação sem interferir com os processos
atualmente executando no sistema de computação, em que a execução da parte da instalação
do conjunto de atualizações inclui a construção    de vários caches de sistema para vários
subconjuntos do conjunto de atualizações em uma área temporária em um dispositivo de
armazenamento legível por computador no sistema de computação, cada cache do sistema
tendo uma hierarquia que espelha a hierarquia de um cache do sistema em uso no sistema de
computação,conforme D1(paragrafo 29) e D2(pagina 16);              enviar uma notificação de
disponibilidade do conjunto de atualizações após executar a parte da instalação do conjunto de
atualizações,conforme D3(paragrafo 31); permitir a seleção de um subconjunto do conjunto de
atualizações para instalação no sistema de computação, o subconjunto do conjunto de
